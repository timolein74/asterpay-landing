Getting Test API Keys

  1. Sign in to Dashboard
    Go to asterpay.io/dashboard and log in
  2. Navigate to Settings ‚Üí API Keys
    Find the API Keys section in your settings
  3. Copy your Test API Key
    Keys starting with sk_test_ are for sandbox
  4. Use in your .env file
    Set ASTERPAY_API_KEY=sk_test_...

Test API Endpoints

Test mode uses the same API endpoints as production:

Endpoint URL
API Base URL https://api.asterpay.io/v1
Create Payment POST /payment_intents
Get Payment GET /payment_intents/:id
Webhooks Same as production

üß™ Test Mode Behavior

When using test API keys, payments are automatically marked as "completed" after a few seconds. No real blockchain transactions occur.

Creating a Test Payment

curl -X POST https://api.asterpay.io/v1/payment_intents \
  -H "Authorization: Bearer sk_test_your_test_key" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": "10.00",
    "currency": "USDC",
    "chain": "bsc",
    "metadata": {
      "user_id": "test_user_123",
      "credits": "100"
    }
  }'

Test Response

{
  "id": "pi_test_abc123",
  "status": "pending",
  "amount": "10.00",
  "currency": "USDC",
  "checkout_url": "https://pay.asterpay.io/test/abc123",
  "test_mode": true
}

Testing Webhooks

Test webhooks work identically to production. Configure your webhook URL in the dashboard and you'll receive events like:

{
  "event": "payment.completed",
  "payment_id": "pi_test_abc123",
  "amount": "10.00",
  "currency": "USDC",
  "test_mode": true,
  "metadata": {
    "user_id": "test_user_123",
    "credits": "100"
  }
}

Testing Locally with ngrok

To test webhooks on localhost:

# Install ngrok
npm install -g ngrok

# Expose your local server
ngrok http 3000

# Use the ngrok URL as webhook URL
# https://abc123.ngrok.io/api/webhook

Test Card Numbers

For testing the checkout flow, use these test wallet addresses:

Scenario Test Address
Successful payment 0x1111...1111 (any valid address)
Payment timeout 0x0000...timeout
Insufficient funds 0x0000...declined

Switching to Live Mode

üü¢ Ready for Production?

Simply replace your test API key (sk_test_) with your live API key (sk_live_) in your environment variables. No code changes required!

# Test mode
ASTERPAY_API_KEY=sk_test_abc123...

# Live mode (just change the key)
ASTERPAY_API_KEY=sk_live_xyz789....env
